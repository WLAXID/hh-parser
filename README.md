## Установка

### Требования

- Python 3.8+

### Установка

```bash
# Клонировать репозиторий
git clone <repository-url>
cd hh-parser

# Создать виртуальное окружение
python -m venv .venv

# Активировать виртуальное окружение
# Linux/macOS:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# Установить пакет в режиме разработки
pip install -e .
```

После установки будет доступна команда `hh-parser`.

## Быстрый старт

```bash
# 1. Авторизоваться
hh-parser authorize your_email@example.com

# 2. Запустить парсинг
hh-parser parse

# 3. Экспортировать результаты
hh-parser export --output employers.csv
```

## Команды

### Авторизация

```bash
hh-parser authorize <username> [опции]
```

Авторизация в системе hh.ru для получения токена доступа.

**Алиасы:** `authenticate`, `auth`, `login`

**Аргументы:**

| Аргумент   | Описание                           |
| ---------- | ---------------------------------- |
| `username` | Email или номер телефона для входа |

**Опции:**

| Опция                     | Описание                               |
| ------------------------- | -------------------------------------- |
| `-p, --password <пароль>` | Пароль для входа                       |
| `-n, --no-headless`       | Показать окно браузера при авторизации |
| `-m, --manual`            | Ручной режим ввода данных              |
| `-k, --use-kitty`         | Вывод капчи в kitty (не реализовано)   |
| `-s, --use-sixel`         | Вывод капчи в sixel (не реализовано)   |

**Примеры:**

```bash
# Авторизация с email
hh-parser authorize user@example.com

# Авторизация с паролем
hh-parser authorize user@example.com --password secret

# Авторизация с отображением браузера
hh-parser authorize user@example.com --no-headless

# Ручной режим (для сложных случаев)
hh-parser authorize --manual
```

### Парсинг работодателей

```bash
hh-parser parse [опции]
```

Сбор данных о работодателях с hh.ru.

**Опции:**

| Опция                   | По умолчанию | Описание                                                  |
| ----------------------- | ------------ | --------------------------------------------------------- |
| `--area <id> [<id>...]` | Все регионы  | Фильтр по региону (ID региона, можно указать несколько)   |
| `--only-with-vacancies` | Выкл.        | Только работодатели с открытыми вакансиями                |
| `--sort-by <метод>`     | `by_name`    | Сортировка: `by_name` или `by_vacancies_open`             |
| `--per-page <число>`    | 100          | Количество результатов на странице (макс. 100)            |
| `--mode <режим>`        | `fast`       | Режим работы (см. ниже)                                   |
| `--resume`              | Выкл.        | Возобновить парсинг, пропуская существующих работодателей |

**Режимы работы (`--mode`):**

| Режим        | Описание                                               |
| ------------ | ------------------------------------------------------ |
| `fast`       | Базовая информация: название, сайт, ссылка на hh.ru    |
| `full`       | Полный парсинг с расчётом среднего количества откликов |
| `stats-only` | Обновление статистики для существующих работодателей   |

**Примеры:**

```bash
# Базовый парсинг всех работодателей
hh-parser parse

# Полный парсинг с расчётом статистики
hh-parser parse --mode full

# Парсинг по конкретным регионам
hh-parser parse --area 1 2

# Только работодатели с вакансиями, сортировка по количеству вакансий
hh-parser parse --only-with-vacancies --sort-by by_vacancies_open

# Возобновить прерванный парсинг
hh-parser parse --resume

# Обновить статистику для существующих работодателей
hh-parser parse --mode stats-only
```

### Экспорт данных

```bash
hh-parser export --output <файл> [опции]
```

Экспорт собранных данных о работодателях.

**Опции:**

| Опция                     | По умолчанию | Описание                                        |
| ------------------------- | ------------ | ----------------------------------------------- |
| `--format <формат>`       | `csv`        | Формат экспорта: `csv` или `json`               |
| `--output <файл>`         | Обязательно  | Путь к выходному файлу                          |
| `--industry <id>`         | Все          | Фильтр по отрасли (ID отрасли)                  |
| `--area <название>`       | Все          | Фильтр по региону (название или часть названия) |
| `--min-vacancies <число>` | 0            | Минимальное количество открытых вакансий        |

**Примеры:**

```bash
# Экспорт в CSV
hh-parser export --output employers.csv

# Экспорт в JSON
hh-parser export --format json --output employers.json

# Экспорт с фильтрацией по количеству вакансий
hh-parser export --output top_employers.csv --min-vacancies 10

# Экспорт работодателей из конкретного региона
hh-parser export --output moscow.csv --area "Москва"

# Экспорт в JSON с фильтрами
hh-parser export --format json --output filtered.json --min-vacancies 5 --area "Москва"
```

### Справочники (отрасли и регионы)

```bash
hh-parser reference <тип> [опции]
```

Вывод справочников из API hh.ru для использования фильтров при парсинге.

**Алиасы:** `ref`, `dict`, `dictionaries`

**Подкоманды:**

| Подкоманда   | Алиас     | Описание            |
| ------------ | --------- | ------------------- |
| `industries` | `ind`     | Справочник отраслей |
| `areas`      | `regions` | Справочник регионов |

**Опции для industries:**

| Опция                | По умолчанию | Описание                                 |
| -------------------- | ------------ | ---------------------------------------- |
| `--format <формат>`  | `table`      | Формат вывода: `table`, `json`, `simple` |
| `--search <строка>`  | —            | Фильтр по названию отрасли               |
| `--page <число>`     | 0            | Номер страницы (начиная с 0)             |
| `--per-page <число>` | 50           | Элементов на странице (0 — показать все) |

**Опции для areas:**

| Опция                 | По умолчанию | Описание                                         |
| --------------------- | ------------ | ------------------------------------------------ |
| `--format <формат>`   | `table`      | Формат вывода: `table`, `json`, `simple`, `tree` |
| `--search <строка>`   | —            | Фильтр по названию региона                       |
| `--country <строка>`  | —            | Фильтр по стране (ID или название)               |
| `--max-depth <число>` | 3            | Максимальная глубина для дерева                  |
| `--page <число>`      | 0            | Номер страницы (начиная с 0)                     |
| `--per-page <число>`  | 50           | Элементов на странице (0 — показать все)         |

**Примеры:**

```bash
# Список всех отраслей (постранично)
hh-parser reference industries

# Следующая страница отраслей
hh-parser reference industries --page 1

# Показать все отрасли без пагинации
hh-parser reference industries --per-page 0

# Поиск отрасли по названию
hh-parser reference industries --search "информ"

# Экспорт отраслей в JSON
hh-parser reference industries --format json > industries.json

# Список регионов России (постранично)
hh-parser reference areas --country "Россия"

# Следующая страница регионов
hh-parser reference areas --country "Россия" --page 1

# Показать все регионы без пагинации
hh-parser reference areas --per-page 0

# Дерево регионов (без пагинации)
hh-parser reference areas --format tree

# Поиск региона по названию
hh-parser reference areas --search "Москва"

# Экспорт регионов в JSON
hh-parser reference areas --format json > areas.json
```

### Информация о пользователе

```bash
hh-parser whoami
```

Показать информацию о текущем авторизованном пользователе.

**Выводимая информация:**

- ID пользователя
- ФИО
- Email
- Телефон
- Тип аккаунта (соискатель/работодатель/администратор)

**Пример:**

```bash
hh-parser whoami
# Вывод:
# Информация о текущем пользователе:
#  ID: 123456
#  Имя: Иван Иванов
#  Email: user@example.com
#  Телефон: +7 (999) 123-45-67
#  Является соискателем: True
#  Является работодателем: False
#  Является администратором: False
```

## Глобальные параметры

Эти параметры можно использовать с любой командой:

| Параметр                    | Описание                                                                                                      |
| --------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `-v, --verbosity`           | Увеличить уровень детализации вывода (можно использовать несколько раз: `-v`, `-vv`, `-vvv`)                  |
| `-c, --config-dir <путь>`   | Путь до директории с конфигурацией                                                                            |
| `--profile-id <id>`         | Используемый профиль (подкаталог в config-dir). Также можно задать через переменную окружения `HH_PROFILE_ID` |
| `-d, --api-delay <секунды>` | Задержка между запросами к API HH                                                                             |
| `--user-agent <строка>`     | User-Agent для запросов                                                                                       |
| `--proxy-url <url>`         | Прокси для запросов и авторизации                                                                             |

**Примеры:**

```bash
# Подробный вывод
hh-parser -vvv parse

# Использование конкретного профиля
hh-parser --profile-id work parse

# Указание директории конфигурации
hh-parser --config-dir /path/to/config parse

# Использование прокси
hh-parser --proxy-url http://proxy:8080 parse

# Настройка задержки между запросами
hh-parser --api-delay 1.5 parse
```

## Конфигурация

### Структура директории конфигурации

По умолчанию все данные (конфигурация, база данных, cookies) сохраняются в директории `data/` внутри проекта:

```
hh-parser/
└── data/
    └── default/           # Профиль по умолчанию
        ├── config.json    # Файл конфигурации
        ├── cookies.txt    # Cookies сессии
        └── hh_parser.db   # База данных SQLite
```

Для использования нескольких профилей:

```
hh-parser/
└── data/
    ├── default/           # Профиль по умолчанию
    │   ├── config.json
    │   ├── cookies.txt
    │   └── hh_parser.db
    └── work/              # Пример другого профиля (--profile-id work)
        ├── config.json
        ├── cookies.txt
        └── hh_parser.db
```

**Приоритет определения директории конфигурации:**

1. Аргумент `--config-dir` / `-c`
2. Переменная окружения `CONFIG_DIR`
3. Директория `data/` внутри проекта (по умолчанию)

### Файл конфигурации (config.json)

```json
{
  "client_id": "your_client_id",
  "client_secret": "your_client_secret",
  "token": {
    "access_token": "...",
    "refresh_token": "...",
    "access_expires_at": "2024-01-01T00:00:00Z"
  },
  "api_delay": 0.5,
  "user_agent": "Mozilla/5.0 ...",
  "proxy_url": "http://proxy:8080"
}
```

### Переменные окружения

| Переменная      | Описание                           |
| --------------- | ---------------------------------- |
| `CONFIG_DIR`    | Путь до директории с конфигурацией |
| `HH_PROFILE_ID` | ID профиля для использования       |
| `HTTP_PROXY`    | HTTP прокси                        |
| `HTTPS_PROXY`   | HTTPS прокси                       |

## Устранение неполадок

### Ошибка "Требуется авторизация"

Выполните команду авторизации:

```bash
hh-parser authorize your_email@example.com
```

### Ошибка "Требуется ввод капчи"

При авторизации может потребоваться ввод капчи. Используйте флаг `--no-headless` для отображения браузера:

```bash
hh-parser authorize your_email@example.com --no-headless
```

### Повреждена база данных

Если база данных повреждена, пересоздайте её:

```bash
hh-parser migrate-db
```

### Проблемы с прокси

Убедитесь, что прокси указан правильно:

```bash
hh-parser --proxy-url http://user:password@proxy:8080 parse
```

Или через переменные окружения:

```bash
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080
hh-parser parse
```
