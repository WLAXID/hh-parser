## Установка

```bash
# Клонировать репозиторий
git clone <repository-url>
cd hh-parser

# Создать виртуальное окружение
python -m venv .venv

# Активировать виртуальное окружение
# Linux/macOS:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# Установить пакет в режиме разработки
pip install -e .

# Установить Playwright браузер (для авторизации)
playwright install chromium
```

После установки будет доступна команда `hh-parser`.

```bash
hh-parser parse

hh-parser parse-contacts
```

## Команды

### Авторизация

```bash
hh-parser authorize <username> [опции]
```

Авторизация в системе hh.ru для получения токена доступа через Playwright.

**Алиасы:** `authenticate`, `auth`, `login`

#### Аргументы

| Аргумент   | Описание                           |
| ---------- | ---------------------------------- |
| `username` | Email или номер телефона для входа |

#### Опции

| Опция                     | Описание                               |
| ------------------------- | -------------------------------------- |
| `-p, --password <пароль>` | Пароль для входа                       |
| `-n, --no-headless`       | Показать окно браузера при авторизации |
| `-m, --manual`            | Ручной режим ввода данных              |

#### Примеры

```bash
# Авторизация с email
hh-parser authorize user@example.com

# Авторизация с паролем
hh-parser authorize user@example.com --password secret

# Авторизация с отображением браузера
hh-parser authorize user@example.com --no-headless

# Ручной режим
hh-parser authorize --manual
```

### Парсинг работодателей

```bash
hh-parser parse [опции]
```

Сбор данных о работодателях с hh.ru.

#### Опции

| Опция                       | По умолчанию           | Описание                                                   |
| --------------------------- | ---------------------- | ---------------------------------------------------------- |
| `-a, --area <id> [<id>...]` | Все регионы            | Фильтр по региону (ID региона, можно указать несколько)    |
| `--only-with-vacancies`     | Выкл.                  | Только работодатели с открытыми вакансиями                 |
| `-s, --sort-by <метод>`     | `by_name` (из конфига) | Сортировка: `by_name` или `by_vacancies_open`              |
| `--per-page <число>`        | 100 (из конфига)       | Количество результатов на странице (макс. 100)             |
| `-m, --mode <режим>`        | `full` (из конфига)    | Режим работы (см. ниже)                                    |
| `-r, --resume`              | Выкл.                  | Возобновить парсинг, пропуская существующих работодателей  |
| `-l, --limit <число>`       | 0                      | Ограничение количества работодателей (0 = без ограничений) |

> **Примечание:** Значения по умолчанию для `--sort-by`, `--per-page` и `--mode` берутся из конфигурационного файла `config.json`. Аргументы CLI имеют приоритет над конфигом.

#### Режимы работы (`--mode`)

| Режим        | Описание                                               |
| ------------ | ------------------------------------------------------ |
| `fast`       | Базовая информация: название, сайт, ссылка на hh.ru    |
| `full`       | Полный парсинг с расчётом среднего количества откликов |
| `stats-only` | Обновление статистики для существующих работодателей   |

#### Примеры

```bash
# Полный парсинг всех работодателей (по умолчанию)
hh-parser parse

# Быстрый парсинг без расчёта статистики
hh-parser parse --mode fast

# Парсинг по конкретным регионам
hh-parser parse --area 1 2

# Только работодатели с вакансиями, сортировка по количеству вакансий
hh-parser parse --only-with-vacancies --sort-by by_vacancies_open

# Ограничить количество работодателей
hh-parser parse --limit 100

# Возобновить прерванный парсинг
hh-parser parse --resume

# Обновить статистику для существующих работодателей
hh-parser parse --mode stats-only
```

### Парсинг контактов

```bash
hh-parser parse-contacts [опции]
```

Извлечение контактной информации (email, телефоны) работодателей из двух источников:

1. **hh.ru API** — из описания компании, вакансий и менеджеров
2. **Сайт работодателя** — парсинг официального сайта с поиском страниц контактов

**Алиасы:** `contacts`

#### Опции

| Опция                       | По умолчанию        | Описание                                                   |
| --------------------------- | ------------------- | ---------------------------------------------------------- |
| `-s, --source <источник>`   | `both` (из конфига) | Источник контактов: `api`, `site` или `both`               |
| `-e, --employer-id <id>...` | Все                 | ID конкретных работодателей для парсинга                   |
| `-l, --limit <число>`       | 0                   | Ограничение количества работодателей (0 = без ограничений) |
| `--site-timeout <сек>`      | 30 (из конфига)     | Таймаут для запросов к сайтам работодателей                |
| `--max-pages <число>`       | 10 (из конфига)     | Максимум страниц для парсинга на одном сайте               |
| `-d, --delay <сек>`         | 2.0 (из конфига)    | Задержка между запросами к одному сайту                    |

> **Примечание:** Значения по умолчанию берутся из конфигурационного файла `config.json`. Аргументы CLI имеют приоритет над конфигом.

#### Алгоритм поиска контактов на сайте

```
1. Парсинг главной страницы сайта
      ↓
2. Поиск ссылок по ключевым словам в anchor text
   ("контакты", "contact", "связаться" и др.)
      ↓
3. Переход только по найденным ссылкам
      ↓
4. Извлечение email и телефонов с найденных страниц
      ↓
5. Дедупликация и нормализация контактов
```

#### Примеры

```bash
# Парсинг контактов для всех работодателей (API + сайт)
hh-parser parse-contacts

# Только из API hh.ru
hh-parser parse-contacts --source api

# Только с сайтов работодателей
hh-parser parse-contacts --source site

# Для конкретных работодателей
hh-parser parse-contacts --employer-id 12345 67890

# С кастомными настройками
hh-parser parse-contacts --site-timeout 60 --max-pages 5 --delay 3.0
```

#### Структура таблицы контактов в БД

| Поле               | Тип      | Описание                                    |
| ------------------ | -------- | ------------------------------------------- |
| `id`               | INTEGER  | Первичный ключ                              |
| `employer_id`      | INTEGER  | ID работодателя (внешний ключ)              |
| `employer_name`    | TEXT     | Название работодателя                       |
| `contact_type`     | TEXT     | Тип контакта: `email` или `phone`           |
| `value`            | TEXT     | Исходное значение контакта                  |
| `source`           | TEXT     | Источник: `api` или `site`                  |
| `source_url`       | TEXT     | URL страницы, где найден контакт            |
| `normalized_value` | TEXT     | Нормализованное значение (для дедупликации) |
| `created_at`       | DATETIME | Дата создания записи                        |

### Экспорт данных

```bash
hh-parser export --output <файл> [опции]
```

Экспорт собранных данных о работодателях.

#### Опции

| Опция                     | По умолчанию | Описание                                        |
| ------------------------- | ------------ | ----------------------------------------------- |
| `--format <формат>`       | `csv`        | Формат экспорта: `csv` или `json`               |
| `--output <файл>`         | Обязательно  | Путь к выходному файлу                          |
| `--area <название>`       | Все          | Фильтр по региону (название или часть названия) |
| `--min-vacancies <число>` | 0            | Минимальное количество открытых вакансий        |

#### Примеры

```bash
# Экспорт в CSV
hh-parser export --output employers.csv

# Экспорт в JSON
hh-parser export --format json --output employers.json

# Экспорт с фильтрацией по количеству вакансий
hh-parser export --output top_employers.csv --min-vacancies 10

# Экспорт работодателей из конкретного региона
hh-parser export --output moscow.csv --area "Москва"
```

### Справочники (отрасли и регионы)

```bash
hh-parser reference <тип> [опции]
```

Вывод справочников из API hh.ru для использования фильтров при парсинге.

**Алиасы:** `ref`, `dict`, `dictionaries`

#### Подкоманды

| Подкоманда   | Алиас     | Описание            |
| ------------ | --------- | ------------------- |
| `industries` | `ind`     | Справочник отраслей |
| `areas`      | `regions` | Справочник регионов |

#### Опции для industries

| Опция                | По умолчанию | Описание                                 |
| -------------------- | ------------ | ---------------------------------------- |
| `--format <формат>`  | `table`      | Формат вывода: `table`, `json`, `simple` |
| `--search <строка>`  | —            | Фильтр по названию отрасли               |
| `--page <число>`     | 0            | Номер страницы (начиная с 0)             |
| `--per-page <число>` | 50           | Элементов на странице (0 — показать все) |

#### Опции для areas

| Опция                 | По умолчанию | Описание                                         |
| --------------------- | ------------ | ------------------------------------------------ |
| `--format <формат>`   | `table`      | Формат вывода: `table`, `json`, `simple`, `tree` |
| `--search <строка>`   | —            | Фильтр по названию региона                       |
| `--country <строка>`  | —            | Фильтр по стране (ID или название)               |
| `--max-depth <число>` | 3            | Максимальная глубина для дерева                  |
| `--page <число>`      | 0            | Номер страницы (начиная с 0)                     |
| `--per-page <число>`  | 50           | Элементов на странице (0 — показать все)         |

#### Примеры

```bash
# Список всех отраслей
hh-parser reference industries

# Показать все отрасли без пагинации
hh-parser reference industries --per-page 0

# Поиск отрасли по названию
hh-parser reference industries --search "информ"

# Экспорт отраслей в JSON
hh-parser reference industries --format json > industries.json

# Дерево регионов
hh-parser reference areas --format tree

# Поиск региона по названию
hh-parser reference areas --search "Москва"
```

## Глобальные параметры

Эти параметры можно использовать с любой командой:

| Параметр                    | Описание                                                                                                      |
| --------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `-c, --config-dir <путь>`   | Путь до директории с конфигурацией                                                                            |
| `--profile-id <id>`         | Используемый профиль (подкаталог в config-dir). Также можно задать через переменную окружения `HH_PROFILE_ID` |
| `-d, --api-delay <секунды>` | Задержка между запросами к API HH                                                                             |
| `--user-agent <строка>`     | User-Agent для запросов                                                                                       |

### Примеры

```bash
# Использование конкретного профиля
hh-parser --profile-id work parse

# Указание директории конфигурации
hh-parser --config-dir /path/to/config parse

# Настройка задержки между запросами
hh-parser --api-delay 1.5 parse
```

## Конфигурация

### Структура директории конфигурации

По умолчанию все данные (конфигурация, база данных, cookies) сохраняются в директории `data/` внутри проекта:

```
hh-parser/
└── data/
    └── default/           # Профиль по умолчанию
        ├── config.json    # Файл конфигурации
        ├── cookies.txt    # Cookies сессии
        └── hh_parser.db   # База данных SQLite
```

Для использования нескольких профилей:

```
hh-parser/
└── data/
    ├── default/           # Профиль по умолчанию
    │   ├── config.json
    │   ├── cookies.txt
    │   └── hh_parser.db
    └── work/              # Пример другого профиля (--profile-id work)
        ├── config.json
        ├── cookies.txt
        └── hh_parser.db
```

#### Приоритет определения директории конфигурации

| Приоритет | Источник                                         |
| --------- | ------------------------------------------------ |
| 1         | Аргумент `--config-dir` / `-c`                   |
| 2         | Переменная окружения `CONFIG_DIR`                |
| 3         | Директория `data/` внутри проекта (по умолчанию) |

### Файл конфигурации (config.json)

Конфигурационный файл содержит настройки для всех команд парсера. Аргументы CLI имеют приоритет над значениями в конфиге.

```json
{
  "token": {
    "access_token": "...",
    "refresh_token": "...",
    "access_expires_at": 1704067200
  },
  "site_parser": {
    "timeout": 30.0,
    "connect_timeout": 10.0,
    "delay_between_requests": 2.0,
    "max_redirects": 5,
    "max_pages_per_site": 10,
    "user_agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36"
  },
  "parse_contacts": {
    "source": "both",
    "site_timeout": 30,
    "max_pages": 10,
    "delay": 2.0
  },
  "parse": {
    "per_page": 100,
    "sort_by": "by_name",
    "mode": "full"
  }
}
```

### Секции конфигурации

| Секция           | Описание                                       |
| ---------------- | ---------------------------------------------- |
| `token`          | Токены авторизации (автоматически обновляются) |
| `site_parser`    | Настройки парсера сайтов работодателей         |
| `parse_contacts` | Настройки команды `parse-contacts`             |
| `parse`          | Настройки команды `parse`                      |

#### Параметры `site_parser`

| Параметр                 | По умолчанию | Описание                                     |
| ------------------------ | ------------ | -------------------------------------------- |
| `timeout`                | 30.0         | Таймаут на чтение ответа (секунды)           |
| `connect_timeout`        | 10.0         | Таймаут на подключение (секунды)             |
| `delay_between_requests` | 2.0          | Задержка между запросами к одному сайту      |
| `max_redirects`          | 5            | Максимум редиректов                          |
| `max_pages_per_site`     | 10           | Максимум страниц для парсинга на одном сайте |
| `user_agent`             | Chrome 120   | User-Agent для запросов                      |

#### Параметры `parse_contacts`

| Параметр       | По умолчанию | Описание                                     |
| -------------- | ------------ | -------------------------------------------- |
| `source`       | `both`       | Источник контактов: `api`, `site` или `both` |
| `site_timeout` | 30           | Таймаут для запросов к сайтам работодателей  |
| `max_pages`    | 10           | Максимум страниц для парсинга на одном сайте |
| `delay`        | 2.0          | Задержка между запросами                     |

#### Параметры `parse`

| Параметр   | По умолчанию | Описание                                       |
| ---------- | ------------ | ---------------------------------------------- |
| `per_page` | 100          | Количество результатов на странице (макс. 100) |
| `sort_by`  | `by_name`    | Сортировка: `by_name` или `by_vacancies_open`  |
| `mode`     | `full`       | Режим работы: `fast`, `full` или `stats-only`  |

### Переменные окружения

| Переменная      | Описание                           |
| --------------- | ---------------------------------- |
| `CONFIG_DIR`    | Путь до директории с конфигурацией |
