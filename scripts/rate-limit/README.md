# Скрипты для тестирования Rate Limits HH API

Эта папка содержит скрипты для тестирования и анализа rate limits API HeadHunter.

## Настройка

1. Скопируйте `.env-example` в `.env`:

```bash
cp scripts/rate-limit/.env-example scripts/rate-limit/.env
```

2. Заполните `.env` файл вашим access token:

```
HH_ACCESS_TOKEN=YOUR_TOKEN_HERE
```

3. Установите `python-dotenv` для автоматической загрузки `.env` (опционально):

```bash
pip install python-dotenv
```

## Скрипты

### test_rate_limits.py

Основной скрипт для тестирования rate limits HH API. Использует Rich для красивого вывода в консоль.

**Команды:**

#### `list` - Показать список эндпоинтов

```bash
python scripts/rate-limit/test_rate_limits.py list
```

#### `test` - Запустить тест rate limits

```bash
# Базовый тест с 100 запросами
python scripts/rate-limit/test_rate_limits.py test

# С указанием количества запросов
python scripts/rate-limit/test_rate_limits.py test --requests 50

# С указанием эндпоинта
python scripts/rate-limit/test_rate_limits.py test --endpoint areas --requests 30

# С токеном авторизации
python scripts/rate-limit/test_rate_limits.py test --token YOUR_TOKEN --requests 50

# С сохранением отчёта
python scripts/rate-limit/test_rate_limits.py test --requests 50 --output report.json
```

**Параметры:**

| Параметр         | Описание                     | По умолчанию |
| ---------------- | ---------------------------- | ------------ |
| `--requests, -n` | Количество запросов          | 100          |
| `--endpoint, -e` | Эндпоинт для тестирования    | vacancies    |
| `--token, -t`    | Access token для авторизации | из .env      |
| `--output, -o`   | Путь для сохранения отчёта   | -            |

#### `find-min` - Найти минимальную безопасную задержку

Тестирует разные задержки от `--start` до `--min` и строит таблицу результатов.

```bash
# Базовый поиск (от 2s до 0.01s)
python scripts/rate-limit/test_rate_limits.py find-min

# С кастомными параметрами
python scripts/rate-limit/test_rate_limits.py find-min \
    --start 5.0 \
    --min 0.05 \
    --requests 20 \
    --threshold 98.0

# С сохранением результатов
python scripts/rate-limit/test_rate_limits.py find-min --output results.json
```

**Параметры:**

| Параметр          | Описание                        | По умолчанию |
| ----------------- | ------------------------------- | ------------ |
| `--endpoint, -e`  | Эндпоинт для тестирования       | vacancies    |
| `--start, -s`     | Начальная задержка (секунды)    | 2.0          |
| `--min, -m`       | Минимальная задержка (секунды)  | 0.01         |
| `--requests, -n`  | Запросов на каждый тест         | 10           |
| `--threshold, -t` | Минимальный процент успеха      | 95.0         |
| `--token`         | Access token для авторизации    | из .env      |
| `--output, -o`    | Путь для сохранения результатов | -            |

## Доступные эндпоинты

| Название         | Путь             | Описание                                  |
| ---------------- | ---------------- | ----------------------------------------- |
| `vacancies`      | `vacancies`      | Поиск вакансий                            |
| `vacancy_detail` | `vacancies/{id}` | Детали вакансии                           |
| `employers`      | `employers/{id}` | Информация о работодателе                 |
| `me`             | `me`             | Информация о пользователе (требует токен) |
| `dictionaries`   | `dictionaries`   | Справочники                               |
| `areas`          | `areas`          | Регионы                                   |
| `industries`     | `industries`     | Индустрии                                 |
| `metro`          | `metro`          | Станции метро                             |

## Известные rate limits HH API

По документации HH API:

- **Поиск вакансий**: ~1000 запросов в час для неавторизованных пользователей
- **С авторизацией**: лимиты могут отличаться
- **При достижении лимита**: возвращается ошибка 400 с `errors[].value = "limit_exceeded"`

## Примечания

- Скрипты используют Rich для красивого вывода в консоль (как в остальном проекте)
- Прогресс-бары показывают ход выполнения тестов
- Результаты можно сохранять в JSON для последующего анализа
- Токен можно передавать через аргумент `--token` или переменную окружения `HH_ACCESS_TOKEN`
